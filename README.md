## Quick Start

### Installation

```bash
# Clone the repository
git clone https://github.com/yourusername/CK.git
cd CK

# Install dependencies
pip install -r requirements.txt

# Install the package
pip install -e .
```

### Basic Usage

```bash
# Start interactive mode (numbered options)
ck -i

# Encrypt a file (creates demoEncrypt.txt)
ck -e demo.txt

# Decrypt a file (creates demoDecrypt.txt)
ck -d demoEncrypt.txt

# Generate file hashes (5 algorithms by default)
ck -h demo.txt

# Analyze file metadata
ck metadata demo.txt
```

### Command Line Options

- `ck -i` or `ck --interactive` - Start interactive mode
- `ck -e <file>` - Encrypt a file
- `ck -d <file>` - Decrypt a file  
- `ck -h <file>` - Generate file hashes
- `ck --algo <algorithm>` - Specify encryption algorithm (aes-128, 3des)

### Examples

#### Encryption
```bash
# Encrypt demo.txt (creates demoEncrypt.txt)
ck -e demo.txt

# Use 3DES algorithm instead of default AES-128
ck -e demo.txt --algo 3des
```

When encrypting, you'll be prompted for a password. The tool automatically:
- Creates an encrypted file with "Encrypt" suffix (demo.txt ‚Üí demoEncrypt.txt)
- Generates a key file (Key_demo.txt)

#### Decryption
```bash
# Decrypt demoEncrypt.txt (creates demoDecrypt.txt)
ck -d demoEncrypt.txt
```

For decryption, you need the key file that was created during encryption.

#### Hashing
```bash
# Generate hashes with 5 algorithms by default
ck -h demo.txt
```

This creates `demoHash.txt` containing:
- MD5
- SHA1
- SHA256
- SHA512
- BLAKE2B

#### Steganography
```bash
# Hide data in image (interactive mode recommended)
ck -i
# Then select option 4
```

#### Metadata Analysis
```bash
# Analyze file metadata
ck metadata demo.txt

# Detailed analysis
ck metadata suspicious_file.exe
```

## Smart File Naming

The tool uses intelligent file naming conventions:

- **Encryption**: `demo.txt` ‚Üí `demoEncrypt.txt`
- **Decryption**: `demoEncrypt.txt` ‚Üí `demoDecrypt.txt`
- **Hashing**: `demo.txt` ‚Üí `demoHash.txt`
- **Key files**: `demo.txt` ‚Üí `Key_demo.txt`

## Supported Algorithms

### Encryption
- **AES-128** (default) - Fast and secure
- **3DES** - Legacy compatibility

### Hashing (All generated by default)
- **MD5** - Fast checksum
- **SHA1** - Legacy support
- **SHA256** - Current standard
- **SHA512** - Enhanced security
- **BLAKE2B** - Modern algorithm
